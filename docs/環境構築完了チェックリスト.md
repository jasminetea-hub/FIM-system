# 環境構築完了チェックリスト

## ✅ セットアップ手順

### 1. Rのインストール
- [ ] Rがインストールされている
- [ ] Rのバージョン確認: `R --version`
- [ ] Rコンソールが起動できる

### 2. R開発環境のインストール
- [ ] **Windows**: Rtoolsがインストールされている
- [ ] **macOS**: Xcode Command Line Toolsがインストールされている
- [ ] **Linux**: r-base-devパッケージがインストールされている

### 3. Rパッケージのインストール
- [ ] `r_api/install_r_packages.R`を実行
- [ ] tidyverseパッケージがインストールされている
- [ ] caretパッケージがインストールされている
- [ ] randomForestパッケージがインストールされている

### 4. Python環境のセットアップ
- [ ] Python 3.8以上がインストールされている
- [ ] `r_api/setup.bat`（Windows）または`r_api/setup.sh`（macOS/Linux）を実行
- [ ] 仮想環境が作成されている（`r_api/venv/`）
- [ ] 依存関係がインストールされている

### 5. 環境の確認
- [ ] `python r_api/check_environment.py`を実行
- [ ] すべてのチェックが✅になっている

### 6. Rモデルの準備
- [ ] `r_api/save_r_models.R`を実行（既に学習済みの場合はスキップ可能）
- [ ] `r_models/`ディレクトリに`.rds`ファイルが存在する

### 7. FastAPIサーバーの起動
- [ ] `python r_api/predict_api_fastapi.py`を実行
- [ ] サーバーが正常に起動する
- [ ] エラーメッセージが表示されない

### 8. 動作確認
- [ ] ブラウザで http://localhost:5000/docs にアクセス
- [ ] Swagger UIが表示される
- [ ] `/health`エンドポイントが動作する
- [ ] `/predict`エンドポイントが動作する

## 📝 次のステップ

環境構築が完了したら：

1. **Node.jsサーバーの設定**
   - `server_with_r_fastapi.js`を使用してNode.jsサーバーを起動
   - FastAPIサーバーと連携する

2. **フロントエンドの確認**
   - フォームから予測APIを呼び出す
   - 結果が正しく表示される

3. **本番環境へのデプロイ**
   - Docker Composeを使用する場合
   - クラウドサービスにデプロイする場合

## 🔧 トラブルシューティング

問題が発生した場合：

1. `r_api/check_environment.py`を実行して環境を確認
2. `r_api/環境構築ガイド.md`のトラブルシューティングセクションを参照
3. エラーメッセージを確認して対処
